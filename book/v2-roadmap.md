# 第 2 版 ロードマップ — 「楽しさ」と「実用」をまず厚く

第 1 版は「Lisp の考え方を身につける → 小さな処理系を作る → Web アプリで遊ぶ」までを一気通貫でなぞる構成でした。一方で `99-bibliography.md` の A.5 で挙げた **本書で触れなかったトピック** は、ライブラリの数だけでも一冊分あります。

第 2 版では、**読者が「明日から使いたい」と思えるトピック** を優先し、マニアックな深掘り(マクロ理論、自作 `#lang`、リーダ拡張)は第 3 版以降にまわします。

> 99-bibliography.md A.5 にあった「第 2 版の重点予定: 言語拡張の深掘り」は方針転換し、本ロードマップに置き換えます。

## 0. 方針

第 2 版の重点は次の 2 軸です。

1. **楽しさ** — 動かして気持ちよい、絵が出る、ゲームが遊べる、グラフが描ける
2. **実用** — DB・HTTP・JSON・CLI ツール配布など、業務や個人プロジェクトで明日から役に立つ

「マニアックさ」(マクロ理論、衛生性、リーダ拡張、コンパイル過程の解読) は第 3 版でまとめて扱います。第 2 版でも 16 章 (マクロ入門) と 17 章 (発展トピック) は維持しますが、読者層を広げる新章を増やすことを優先します。

## 2. スタイル方針 — 「14 章よりもっと平易に」

第 14 章 (Mini Lisp) は読者好評だが、**「2-3 章前のルールを忘れていると詰まる」** という読者を想定して以下を強化する。

### 2.1 章頭の「30 秒復習」ボックス
各章冒頭に、その章で頻出する **過去ルールの 5–10 行復習** を必ず置く。

```markdown
> **30 秒復習**
> - `hash-ref h k default-thunk` — キーが無いとき thunk が呼ばれる(第 9 章)
> - `for/list` — 各反復値を集めてリストにする(第 8 章)
> - `match` の `(? 述語)` — 述語が真ならマッチ(第 9 章)
```

### 2.2 コードブロックを 5–15 行ずつ刻む
14 章はブロックがやや長い (20+ 行)。第 2 版では **小刻み + 直後に行注釈表** を基本パターンにする。

| 行 | コード | 解説 |
|----|--------|------|
| 1 | `(define conn (sqlite3-connect ...))` | 接続オブジェクトを 1 度作って使い回す |

### 2.3 「Python ならこう」対比を毎章 1 箇所
00-preface.md の方針を全章で徹底。例:

```text
Python: requests.get(url).json()
Racket: (response-json (get url))
```

### 2.4 章末の「API ミニ辞典」
新登場 API を **章末に再掲表**(関数名 / シグネチャ / 1 行説明)。読み返しのインデックスとして機能する。

### 2.5 図を毎章 1–2 枚
Mermaid のシーケンス図、データ流、状態遷移、テーブルスキーマ図など。
DB 章なら「リクエスト → prepare → execute → fetch」のシーケンス図、ゲーム章なら big-bang のイベントループ図など。

### 2.6 「復習」コラム(本文の流れを止めずに思い出させる)
本文中で過去の細かいルールを使うときは、引用ブロックで **その場再説明**:

```markdown
> **復習**: `(? procedure?)` は match の述語パターン。第 9 章で扱った。
```

### 2.7 「ハマりどころ」セクションを必ず置く
14.9 のような節を **5 件以上** で必ず用意。読者が詰まる場所はだいたい同じなので予防射撃する。

### 2.8 章末に「次章への橋渡し」
1–2 行で「次章ではこれを使ってこういうものを作る」と書き、シリーズ感を出す。

## 3. 章ごとのテンプレ

```markdown
# 第 N 章 タイトル

> **30 秒復習**
> - 過去ルール 1
> - 過去ルール 2

## N.1 何ができるようになるか
完成形のスクリーンショット / REPL 出力を 1 つ。

## N.2 必要なライブラリと前提
`require` 文、`raco pkg install` 手順、対応 Racket バージョン。

## N.3 最小サンプル(動かして気持ちよくなる)
20 行以内で動くコード + 実行結果。

## N.4 構造の俯瞰
Mermaid 図 1 枚。

## N.5–N.x 機能ごとの詳細
コード小刻み + 行注釈表。「Python ならこう」 1 箇所。

## N.x+1 ハマりどころ
最低 5 件。

## N.x+2 拡張のヒント
易しめ・難しめ・挑戦の 3 段階。

## N.x+3 まとめ
箇条書き 5–8 個。

## API ミニ辞典
表形式で再掲。

## 手を動かしてみよう
ヒント・部分解答・完全解答を solutions/chNN.md に。

## 次章への橋渡し
1–2 行。
```

## 5. 開発の進め方

1. **執筆順**: 21 章 (DB) → 22 章 (HTTP/JSON) → 18 章 (ゲーム) を最初の 3 本に
   - 「実用 2 本でリピーター獲得 → ゲーム章でバズ」の順
2. 各章は `examples/chNN/`, `solutions/chNN.md` を必ずペアで用意
3. 1 章 = 250–400 行を目安(14 章のスタイルを継続、ただしブロックは細かく)
4. ワークフロー: 章草稿 PR → セルフレビュー → サンプルコードを `raco test` → マージ
5. 各章 PR は **コード例を必ず実機実行** し、出力を本文に貼る (00-preface.md の方針)

## 6. やらないことの宣言

第 2 版では **以下は扱わない** と明示する(第 3 版以降の領域):

- マクロ衛生性の理論的解説(束縛 / スコープ / 相)
- 自作 `#lang` とリーダ拡張
- `expand` / `compile` の内部
- `chaperone` / `impersonator` の詳細
- Typed Racket の Refinement / Occurrence Typing 詳説

これらは第 3 版「言語を作る言語編」として独立して扱う構想。

参考までに、第 2 版完了後に着手予定:
