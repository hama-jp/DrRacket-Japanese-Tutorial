## 問題 1. 5 つの式の出力を予想して確認する

### ヒント

評価規則を 1 つずつ当てはめる:

- `(+ 1 2 3)` は関数適用
- `(* (+ 1 2) (- 10 5))` は内側から評価
- `(if ...)` は **特殊形式**、`'big` / `'small` はシンボル
- `(quote ...)` は中身を **評価せず** 返す
- 準クォート ``` ` ``` / `,` / `,@` はテンプレート埋め込み

### 部分解答

| 式 | 予想される値 |
| --- | --- |
| `(+ 1 2 3)` | `6` |
| `(* (+ 1 2) (- 10 5))` | `15` |
| `(if (> 3 5) 'big 'small)` | `'small` |
| `(quote (if (> 3 5) 'big 'small))` | `'(if (> 3 5) (quote big) (quote small))` |
| `` `(1 ,(+ 1 1) ,@(list 3 4) 5) `` | `'(1 2 3 4 5)` |

### 完全解答

```text
> (+ 1 2 3)
6
> (* (+ 1 2) (- 10 5))
15
> (if (> 3 5) 'big 'small)
'small
> (quote (if (> 3 5) 'big 'small))
'(if (> 3 5) 'big 'small)
> `(1 ,(+ 1 1) ,@(list 3 4) 5)
'(1 2 3 4 5)
```

4 番目の表示 `'(if (> 3 5) 'big 'small)` は、内側の `'big` / `'small` を REPL が省略表示で `'foo` と書いているだけで、実体は `(quote big)` / `(quote small)` を要素に持つリストです。`(list? (quote (if (> 3 5) 'big 'small)))` が `#t` になることで S 式が **ただのリスト** だとわかります。

5 番目は **準クォート** の典型例。

- `` ` `` で囲むと中身は基本そのまま記号扱い
- `,x` は「その場で `x` を評価して埋める」
- `,@xs` は「評価結果のリストを **展開** して埋める」

## 問題 2. `(1 2 3)` はなぜエラーになるか

### ヒント

`'(1 2 3)` ならリストですが、クォートが **無い** と Racket は「先頭を関数、残りを引数とみなして呼び出す」 と解釈します。

### 部分解答

「`1` を関数として呼んだ」と言われます。数値 `1` は手続きではないので契約違反。

### 完全解答

```text
> (1 2 3)
; application: not a procedure;
;  expected a procedure that can be applied to arguments
;   given: 1
;   arguments...:
;    2
;    3
```

同じ理由で `("hello" 1 2)` などもエラー。評価規則が「先頭要素の値を関数として呼ぶ」であることが、この失敗を通して腹落ちします。

期待される書き方:

```racket
(list 1 2 3)     ; => '(1 2 3)
'(1 2 3)         ; => '(1 2 3)(quote と同義)
```

## 問題 3. シンボル同士を `eq?` と `equal?` で比較

### ヒント

Racket のシンボルは **インターン** されています。つまり、同じ綴りのシンボルはメモリ上でも同一オブジェクト。

### 部分解答

```racket
(eq?    (string->symbol "hi") 'hi) ; => #t
(equal? (string->symbol "hi") 'hi) ; => #t
(eq?    'hi 'hi)                   ; => #t
(eq?    "hi" "hi")                 ; => 環境依存(#t または #f)
(equal? "hi" "hi")                 ; => #t
```

文字列同士の `eq?` は環境依存で、シンボルは常に `eq?` で一致します。

### 完全解答

```text
> (eq? (string->symbol "hi") 'hi)
#t
> (equal? (string->symbol "hi") 'hi)
#t
> (eq? 'hi 'hi)
#t
> (symbol->string 'hello)
"hello"
> (string->symbol "hello")
'hello
```

仕上げに意味を整理:

- `eq?` — ポインタ比較(同じオブジェクトか)
- `eqv?` — `eq?` に近いが、数値やキャラクタを値で比較
- `equal?` — 構造まで潜って値で比較

シンボルは `eq?` で比較して OK、文字列やリストは `equal?` が基本です。

この「シンボルは軽量でインターンされた識別子」という性質が、のちのパターンマッチやマクロの土台になります。
