# 第 2 章 解答

> 対象: [book/02-setup.md §手を動かしてみよう](../book/02-setup.md)

## 問題 1. `greet` を DrRacket で動かす

### ヒント

問題文のコードをそのまま貼り付けて `Run`。REPL に何が表示されるかを注意深く見てください。

### 部分解答

REPL には最後の式の値が **ダブルクォート付き** で出力されます。

```text
"こんにちは、レキさん!"
```

`define` 自体は値を表示しないので、REPL に出るのは最終行 `(greet "レキ")` の結果だけです。

### 完全解答

DrRacket で `Run` した直後の REPL は次のようになります(プロンプト `>` は DrRacket の入力待ち)。

```text
Welcome to DrRacket, version 8.10 [cs].
Language: racket, with debugging; memory limit: 128 MB.
"こんにちは、レキさん!"
>
```

本文で述べた **「REPL は値を読み戻せる形(read できる形)で表示する」** という性質が、文字列がクォート付きで出ていることから確認できます。

## 問題 2. Check Syntax で束縛の矢印を見る

### ヒント

画面上部の `Check Syntax` ボタン(DrRacket によっては虫眼鏡アイコン)を押すと、静的解析が走って識別子どうしが矢印で結ばれます。

### 部分解答

`greet` の引数 `name` の位置と、`string-append` の第 2 引数として使われている `name` の位置が、青または紫色の矢印で結ばれるはずです。

### 完全解答

- 矢印は **「定義元 → 使用箇所」** の向きに付きます。
- `greet` の上にマウスを重ねると、呼び出し箇所 `(greet "レキ")` へも矢印が延びます。
- `string-append` はコアライブラリ由来なので、矢印は出ず識別子が青くハイライトされるだけです。

もし `name` の矢印が出ない場合は、`#lang racket` 行の有無と `define` の綴りを疑ってください。

## 問題 3. ターミナルから `racket greet.rkt` を実行する

### ヒント

コードをファイルとして保存し(ここでは `greet.rkt` とします)、ターミナルで `racket greet.rkt` を叩きます。DrRacket の REPL とは表示が微妙に違うはずです。

### 部分解答

ターミナルでは、`(greet "レキ")` 式の結果が **`print` ではなく `writeln` 相当の形式** で表示されます。REPL との主な差は、ファイル実行ではトップレベルの式の値を自動表示しない `racket` のデフォルトに対し、**`#lang racket` が最終式を `print` する** 仕組みがあるため、結局ダブルクォート付きで出ます。

```bash
$ racket greet.rkt
"こんにちは、レキさん!"
```

### 完全解答

```bash
$ cat greet.rkt
#lang racket
(define (greet name)
  (string-append "こんにちは、" name "さん!"))
(greet "レキ")

$ racket greet.rkt
"こんにちは、レキさん!"
```

DrRacket の REPL に出たのと同じく、文字列はダブルクォート付きで出ます。両者の違いが浮かび上がるのは第 3 章で扱う `display` / `write` / `print` の話に進んだ時で、たとえば `(display (greet "レキ"))` ならクォートが取れて `こんにちは、レキさん!` だけが出ます。
