## 問題 1. `print` / `write` / `display` / `printf` の違い

### ヒント

3 つの関数の責務を切り分けて覚えると、以後の出力制御が一気に楽になります。

- `print` — Racket REPL が最終式を表示するのと同じ形(**再読み込み可能**)
- `write` — データを **`read` が読み戻せる** 形(ほぼ `print` と同じ)
- `display` — **人間向け**。クォートや文字ラベルを外して本文だけを出す
- `printf` — C の `printf` 相当。`~a` は `display` 風、`~s` は `write` 風

### 部分解答

```racket
(print "☕")   ; ⇒ "☕"
(write "☕")   ; ⇒ "☕"
(display "☕") ; ⇒ ☕
(printf "~a / ~s~n" "☕" "☕")
; ⇒ ☕ / "☕"
```

`~n` は改行。`~a` と `~s` の対比が本問の見どころです。

### 完全解答

DrRacket の REPL で順に実行した結果:

```text
> (print "☕")
"☕"
> (write "☕")
"☕"
> (display "☕")
☕
> (printf "~a / ~s~n" "☕" "☕")
☕ / "☕"
```

- `print` / `write` は同じ文字列を返しており、どちらも **プログラムとして書き戻せる** 表現
- `display` は人間が読む用
- `printf` の `~a` は `display` 風、`~s` は `write` 風

「ログに出したいのか、保存して再読込したいのか」で使い分けを決めると迷いません。

## 問題 2. `area` を別ファイルで定義して REPL から呼ぶ

### ヒント

`area.rkt` として保存し、DrRacket なら `Run` で REPL に取り込まれます。ターミナルなら `racket -i -f area.rkt` で対話モードに入れます。

### 部分解答

```racket
#lang racket
(define pi 3.14159265358979)
(define (area r) (* pi r r))
```

REPL で `(area 10)` と打つ:

```text
> (area 10)
314.159265358979
```

### 完全解答

DrRacket:

1. Definitions Window に上記コードを貼って `Ctrl+S` で保存
2. `Run` を押す
3. REPL で `(area 10)` 、`(area 2.5)` を確認

```text
> (area 10)
314.159265358979
> (area 2.5)
19.634954084936204
```

ターミナル:

```bash
$ racket -i -f area.rkt
Welcome to Racket v8.10 [cs].
> (area 10)
314.159265358979
```

## 問題 3. 意図的にエラーを出す — `(+ 1 "two")`

### ヒント

Racket のエラーメッセージは **上から読む** のが原則です。関数名、期待された型、実際に渡された値、発生箇所の順で書かれます。

### 部分解答

```text
+: contract violation
  expected: number?
  given: "two"
  argument position: 2nd
```

- 契約違反を出したのは `+`
- `number?` を期待していた
- 実際には `"two"` が渡された
- それは 2 番目の引数だった

### 完全解答

```text
> (+ 1 "two")
; +: contract violation
;   expected: number?
;   given: "two"
;   argument position: 2nd
;   other arguments...:
;    1
```

この形式に慣れると、以降のエラーを読むのが一気に楽になります。DrRacket では該当箇所に赤いアンダーラインが入り、`Check Syntax` 前でも問題の式をクリックで追えます。

参考: `string->number` で変換してから足すのが定番の直し方です。

```racket
(+ 1 (or (string->number "two") 0))
;; => 1  ; "two" は数値化できないので 0 が足される
```
