## 問題 1. `fib-iter` に rackunit テストを 5 件以上

### ヒント

境界値(`0`, `1`)を必ず入れ、中間値を 2-3 件。さらに **性質テスト**(`F(n+2) = F(n+1) + F(n)`)を `for` で回すと、少ない記述で網羅感が出ます。

### 部分解答

```racket
(module+ test
  (require rackunit)
  (check-equal? (fib-iter 0)  0)
  (check-equal? (fib-iter 1)  1)
  (check-equal? (fib-iter 7)  13)
  (check-equal? (fib-iter 10) 55)
  (check-equal? (fib-iter 20) 6765))
```

### 完全解答

完全版は [`solutions/ch12/fib-tests.rkt`](ch12/fib-tests.rkt)。実行:

```bash
$ raco test solutions/ch12/fib-tests.rkt
raco test: (submod "solutions/ch12/fib-tests.rkt" test)
36 tests passed
```

要点:

- **境界**: `(fib-iter 0)` / `(fib-iter 1)` は特別扱い。
- **既知値**: ネット上の早見表や OEIS の **A000045** でスポットチェック。
- **性質テスト**: `F(n+2) = F(n+1) + F(n)` を `for` で回すと、テスト件数を稼ぎつつ「関数の定義そのもの」を検証できます。これは **プロパティベーステスト** の初歩で、`rackcheck` などのライブラリを使えばさらにランダム入力まで広げられます。

## 問題 2. `divide` を `->i` 契約で包む

### ヒント

`->i` は **引数同士の依存** や **結果値への制約** を書ける contract。

- `[b (and/c number? (not/c zero?))]` で 0 除算を止める
- `[result (a b) ...]` で結果側の条件を書けば ポストコンディション

### 部分解答

```racket
(provide (contract-out
          [divide (->i ([a number?]
                        [b (and/c number? (not/c zero?))])
                      [result (a b) (and/c number?
                                           (lambda (r) (>= r 0)))])]))

(define (divide a b) (/ a b))
```

### 完全解答

完全版: [`solutions/ch12/divide.rkt`](ch12/divide.rkt)

テスト実行:

```bash
$ raco test solutions/ch12/divide.rkt
raco test: (submod "solutions/ch12/divide.rkt" test)
6 tests passed
```

挙動:

- `(divide 10 2)` → `5`
- `(divide 10 0)` → **契約違反**(`b` の `(not/c zero?)` で弾かれる)
- `(divide 10 "two")` → **契約違反**(`b` の `number?` で弾かれる)
- `(divide -10 2)` → **ポストコンディション違反**(結果が負)

ポストコンディション違反のエラーは次のようになります。

```text
divide: broke its own contract
  promised: (and/c number? (lambda (r) (>= r 0)))
  produced: -5
  in: the range of
      (->i
       ((a number?)
        (b (and/c number? (not/c zero?))))
       (result
        (a b)
        (and/c number? (lambda (r) (>= r 0)))))
```

「誰の責任か」がメッセージから読み取れます。

- **promised / produced** が出る → **関数側の契約違反**(divide の実装が約束を破った)
- **broke (a contract on arguments)** → **呼び出し側の契約違反**

この **責任の割り振り** が `racket/contract` の最大の強みで、境界を明示すると後から触る人が「自分が信用してよい部分」「疑うべき部分」を迷わなくなります。

## 問題 3. `string-join` の契約

### ヒント

まず「どんな関数か」を思い出します。

```racket
(string-join '("a" "b" "c") "-")
;; => "a-b-c"
```

- 第 1 引数: **文字列のリスト**
- 第 2 引数: **区切り文字列**(省略時は `" "`)
- 結果: **文字列**

さらに実はキーワード引数がある(先頭 `#:before-first`、末尾 `#:after-last`、ステップ間 `#:before-last`)ので、真面目に書くならそれらも含めます。

### 部分解答

最小の契約(位置引数のみ):

```racket
(require racket/string)

(provide (contract-out
          [string-join (->* ((listof string?))
                            (string?)
                            string?)]))
```

`->*` は「必須引数のリスト / 省略可引数のリスト / 結果」の順で書きます。位置引数なら `->` でも十分ですが、省略可があるので `->*` を使います。

### 完全解答

キーワード引数まで含めた精度の高い契約:

```racket
(require racket/string)

(provide (contract-out
          [string-join
           (->* ((listof string?))                          ; 必須
                (string?                                    ; 省略可: 区切り
                 #:before-first string?
                 #:before-last  string?
                 #:after-last   string?)
                string?)]))                                 ; 結果
```

自分で書いて動作確認する流儀は本文通り:

```racket
#lang racket
(require racket/string)

(provide (contract-out
          [string-join
           (->* ((listof string?))
                (string?
                 #:before-first string?
                 #:before-last  string?
                 #:after-last   string?)
                string?)]))

(module+ test
  (require rackunit)
  (check-equal? (string-join '("a" "b" "c") "-") "a-b-c")
  (check-equal? (string-join '()) "")
  (check-exn exn:fail:contract?
             (lambda () (string-join '("a" 1) "-"))))
```

重要な観察:

- **リストの要素型** を `(listof string?)` で絞る。Racket の contract は **遅延検査** なので、要素まで見る場合は `string-join` が実際にリストを走査したときに判定が走ります。
- 省略可の区切り引数は **既定値** を書かないので、`string?` だけで十分。
- 結果 `string?` を **必ず** 書く習慣にしておくと、誤って `void` を返すような実装ミスを早期に弾けます。

発展トピックとして、`contract` と `list/c` の違いに触れておきます。

```racket
(listof string?)          ; 任意長の同質リスト
(list/c string? string?)  ; 長さ固定の異質リスト(string × string)
(cons/c string? string?)  ; 先頭 string・残り string の cons
```

`string-join` は任意長を受けるので `listof` を使います。
