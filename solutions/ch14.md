## 問題 1. `cond` を追加する

### ヒント

問題文のテンプレートをそのまま `mini-eval` の節に追加すればよい。ポイントは次の順で節を試すこと。

1. 節リストが空 → `(void)` を返す
2. 節の先頭が `'else` → 本体を評価
3. 条件を評価して真なら本体を評価
4. それ以外は次の節へ

### 部分解答

```racket
[(list* 'cond clauses)
 (let loop ([cs clauses])
   (cond
     [(null? cs) (void)]
     [(eq? (caar cs) 'else)      (eval-body (cdar cs) e)]
     [(mini-eval (caar cs) e)    (eval-body (cdar cs) e)]
     [else (loop (cdr cs))]))]
```

`eval-body` は元の mini-lisp に既に定義済み。これを `mini-eval` の他の節の近くに足すだけ。

### 完全解答

[`solutions/ch14/mini-lisp.rkt`](ch14/mini-lisp.rkt) の `mini-eval` 内を参照。確認:

```bash
$ raco test solutions/ch14/mini-lisp.rkt
raco test: (submod "solutions/ch14/mini-lisp.rkt" test)
3 tests passed
```

テストの 1 件目が、問題文に書かれている `classify` のサンプルを動かしています。

## 問題 2. `set!` を追加する

### ヒント

問題文で示唆されているとおり、局所フレームを **書き換え可能** にする必要がある。

- フレーム = `(mcons name value)` のリスト(本文の `(cons name value)` の連想リスト相当)
- `mcar` / `mcdr` / `set-mcdr!` を使って値を更新
- ルックアップと更新で共通の「セルを探す関数」を 1 つ用意するとスッキリする

### 部分解答

セル探索 + `set!` 節:

```racket
(define (find-cell frames sym)
  (cond
    [(null? frames) #f]
    [else
     (define hit (findf (lambda (c) (eq? (mcar c) sym)) (car frames)))
     (or hit (find-cell (cdr frames) sym))]))

(define (env-set! e sym value)
  (define c (find-cell (env-frames e) sym))
  (cond
    [c (set-mcdr! c value)]
    [(hash-has-key? (env-globals e) sym)
     (hash-set! (env-globals e) sym value)]
    [else (error 'set! "unbound: ~v" sym)]))

;; mini-eval の match 節に追加
[(list 'set! (? symbol? name) v-expr)
 (env-set! e name (mini-eval v-expr e))
 (void)]
```

### 完全解答

フレームの作り方も `mcons` ベースに変える必要があります。添付コードの要所:

```racket
(define (extend-env e names values)
  (define frame (map (lambda (n v) (mcons n v)) names values))
  (env (cons frame (env-frames e)) (env-globals e)))

(define (env-lookup e sym)
  (define c (find-cell (env-frames e) sym))
  (cond
    [c (mcdr c)]
    [else (hash-ref (env-globals e) sym
                    (lambda () (error 'lookup "unbound: ~v" sym)))]))
```

テストが `set!` をちゃんと動かしていることを確認:

```racket
(run '((define (make-counter)
         (let ([n 0])
           (lambda ()
             (set! n (+ n 1))
             n)))
       (define c (make-counter))
       (list (c) (c) (c))))
;; => '(1 2 3)
```

`make-counter` の呼び出しで作られたクロージャが `n` の **可変セル** を共有しているので、3 回呼ぶと `1, 2, 3` と変化します。`set!` が取り込めると、mini-lisp 内でオブジェクト風のものが書けるようになります。

## 問題 3. `compose` を mini-lisp で動かす

### ヒント

本体はすでに mini-lisp が「関数を値として扱える」設計なので、追加実装は不要。問題文のプログラムを `run` に渡すだけで通るはず。

### 部分解答

```racket
(run '((define (compose f g) (lambda (x) (f (g x))))
       (define (inc x) (+ x 1))
       (define (sq  x) (* x x))
       ((compose sq inc) 4)))
;; => 25
```

### 完全解答

テストに同梱済み:

```racket
(check-equal?
 (run '((define (compose f g) (lambda (x) (f (g x))))
        (define (inc x) (+ x 1))
        (define (sq  x) (* x x))
        ((compose sq inc) 4)))
 25)
```

感動ポイント:

- `compose` の戻り値は **クロージャ**(mini-lisp 上では `(closure params body captured-env)` 形式)
- それを `((compose sq inc) 4)` で呼び出している。つまり「関数を返す関数を、呼び出し側でもう一度関数適用している」
- これが自分で書いた 100 行弱の評価器の上で動く、というのが本章の見せ場

第 16 章以降でマクロを学ぶと、この `compose` をマクロとして書き直す道も開けます。
