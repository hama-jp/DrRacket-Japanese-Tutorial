## 問題 1. `/sum?a=1&b=2` → JSON で `3` を返す

### ヒント

`json` ライブラリの `response/jsexpr` を使うと、ハッシュをそのまま JSON に落とせます。キーは **シンボル**(`hasheq`)。

### 部分解答

`hello-web.rkt` の `dispatch` に節を足すだけ:

```racket
[(equal? path '("sum"))
 (define a (string->number (or (extract-binding/single 'a bindings) "")))
 (define b (string->number (or (extract-binding/single 'b bindings) "")))
 (response/jsexpr (hasheq 'result (+ a b)))]
```

### 完全解答

完全版は [`solutions/ch15/hello-web.rkt`](ch15/hello-web.rkt)。エラー処理込みで:

```racket
[(equal? path '("sum"))
 (define a (string->number (or (extract-binding/single 'a bindings) "")))
 (define b (string->number (or (extract-binding/single 'b bindings) "")))
 (cond
   [(and (number? a) (number? b))
    (response/jsexpr (hasheq 'a a 'b b 'result (+ a b)))]
   [else
    (response/jsexpr #:code 400
                     (hasheq 'error "a and b must be numbers"))])]
```

動作確認:

```bash
$ racket solutions/ch15/hello-web.rkt &
$ curl -s "http://127.0.0.1:8000/sum?a=1&b=2"
{"a":1,"b":2,"result":3}
$ curl -s -o /dev/null -w "%{http_code}\n" "http://127.0.0.1:8000/sum?a=x"
400
```

`hasheq` はシンボルキー用のハッシュ(= JSON オブジェクト)。`json` ライブラリが内部で再帰的にトランスレートするので、値側にリストやハッシュを含めることもできます。

## 問題 2. 完了フラグ付き TODO

### ヒント

- データ構造を `id -> (hash 'text 'done)` に変える
- 描画時は `done` が真なら `<s>...</s>` で取り消し線
- `POST /toggle` で `done` を反転

問題文の `id -> (list text done?)` でも動きますが、拡張しやすいので連想ハッシュにしておくと後が楽です(本解答ではそちらを採用)。

### 部分解答

トグル処理の核:

```racket
(define (toggle-todo! id)
  (define cur (hash-ref todos id #f))
  (when cur
    (hash-set! todos id
               (hash-set cur 'done (not (hash-ref cur 'done #f))))))
```

描画:

```racket
(define label (if done `(s ,text) text))
```

### 完全解答

添付の [`solutions/ch15/todo-web.rkt`](ch15/todo-web.rkt) に `POST /toggle` を足して動かしています。動作確認:

```bash
$ racket solutions/ch15/todo-web.rkt &
$ curl -sL -X POST -d 'text=牛乳を買う' http://127.0.0.1:8001/add > /dev/null
$ curl -sL -X POST -d 'id=1'          http://127.0.0.1:8001/toggle > /dev/null
$ curl -s http://127.0.0.1:8001/ | grep -o '<s>.*</s>'
<s>牛乳を買う</s>
```

## 問題 3. `/tmp/todos.json` への永続化

### ヒント

- 起動時: `file-exists?` → あれば `call-with-input-file` + `read-json`
- 更新時: 追加 / 削除 / 反転ごとに `call-with-output-file` + `write-json`

ハッシュの `id` キーは **数値** のままだと JSON にそのまま落ちないので、**配列として保存** するのが無難(各要素が `hasheq 'id 'text 'done`)。

### 部分解答

```racket
(define TODO-FILE "/tmp/todos.json")

(define (load-todos!)
  (when (file-exists? TODO-FILE)
    (define parsed (call-with-input-file TODO-FILE read-json))
    (for ([item (in-list parsed)])
      (hash-set! todos (hash-ref item 'id) item))))

(define (save-todos!)
  (call-with-output-file TODO-FILE #:exists 'replace
    (lambda (out)
      (write-json (for/list ([(_ v) (in-hash todos)]) v) out))))
```

### 完全解答

全体が噛み合った完全版は [`solutions/ch15/todo-web.rkt`](ch15/todo-web.rkt)。主な工夫:

- 起動時に `load-todos!` を 1 回、更新系の関数は終わりに `save-todos!` を呼ぶ。
- `load-todos!` は JSON の読み取りに失敗した時に落ちないよう `with-handlers [exn:fail?]` で包む。
- 読み込んだ `id` の最大値で `next-id` を更新しておかないと、再起動後にプリミティブキーが衝突する。

動作確認(永続化):

```bash
$ rm -f /tmp/todos.json
$ racket solutions/ch15/todo-web.rkt &
$ curl -sL -X POST -d 'text=牛乳'  http://127.0.0.1:8001/add > /dev/null
$ kill %1
$ cat /tmp/todos.json
[{"id":1,"text":"牛乳","done":false}]

$ racket solutions/ch15/todo-web.rkt &   # 再起動
$ curl -s http://127.0.0.1:8001/ | grep 牛乳
```

2 回目の起動でも `牛乳` が表示されれば成功。これで「プロセスを落としても消えない」最低限の永続性が手に入ります。

本番運用したいときの先の選択肢は、本文にもある通り:

- `db` パッケージで SQLite に置き換える(同時書き込みや検索が一気に楽)
- 書き込み頻度が高いならメモリ上のダーティフラグを見て `save-todos!` をデバウンス
- 認証や複数ユーザが絡むとセッション管理 / CSRF 対策が必要
