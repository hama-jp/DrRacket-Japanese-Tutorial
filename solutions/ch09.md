## 問題 1. `book` 構造体と価格の合計

### ヒント

`#:transparent` を付けると REPL で中身が素直に表示されます。合計は `for/sum` と `foldl` の 2 通り書くのがこの問題の狙い。

### 部分解答

```racket
(struct book (title author price) #:transparent)

(define library
  (list (book "SICP"       "Abelson/Sussman" 5800)
        (book "Little Schemer" "Friedman"    2800)
        (book "On Lisp"    "Graham"          4500)
        (book "Let Over Lambda" "Hoyte"      6200)
        (book "HtDP"       "Felleisen"       3600)))

(for/sum ([b (in-list library)]) (book-price b))
;; => 22900
```

### 完全解答

```racket
#lang racket

(struct book (title author price) #:transparent)

(define library
  (list (book "SICP"            "Abelson/Sussman" 5800)
        (book "Little Schemer"  "Friedman"        2800)
        (book "On Lisp"         "Graham"          4500)
        (book "Let Over Lambda" "Hoyte"           6200)
        (book "HtDP"            "Felleisen"       3600)))

;; 書き方 1: for/sum
(define total-for
  (for/sum ([b (in-list library)]) (book-price b)))

;; 書き方 2: foldl
(define total-fold
  (foldl + 0 (map book-price library)))

total-for  ; => 22900
total-fold ; => 22900
(= total-for total-fold) ; => #t
```

`for/sum` は「合計を集める」ことが目的なら一番読みやすい。`foldl` は抽象度が高く、合計以外(積、最大、最小、連結 …)にも同じパターンで使えます。

## 問題 2. 図形の面積と外周

### ヒント

本文でひな型が示されています。あとは REPL で試して、整数 / 実数の扱いを観察しましょう。

### 部分解答

```racket
(struct circle   (r)     #:transparent)
(struct rect     (w h)   #:transparent)
(struct triangle (a b c) #:transparent)

(define (info s)
  (match s
    [(circle r)       (list (* 3.14 r r) (* 2 3.14 r))]
    [(rect w h)       (list (* w h) (* 2 (+ w h)))]
    [(triangle a b c)
     (define s0 (/ (+ a b c) 2))
     (list (sqrt (* s0 (- s0 a) (- s0 b) (- s0 c)))
           (+ a b c))]))
```

### 完全解答

```racket
#lang racket
(require racket/match)

(struct circle   (r)     #:transparent)
(struct rect     (w h)   #:transparent)
(struct triangle (a b c) #:transparent)

;; `pi` を Racket 組み込みに置き換えて精度を上げた版
(define (info s)
  (match s
    [(circle r)
     (list (* pi r r) (* 2 pi r))]
    [(rect w h)
     (list (* w h) (* 2 (+ w h)))]
    [(triangle a b c)
     (define s0 (/ (+ a b c) 2))
     (list (sqrt (* s0 (- s0 a) (- s0 b) (- s0 c)))
           (+ a b c))]))

(info (circle 10))             ; => '(314.1592653589793 62.83185307179586)
(info (rect 3 4))              ; => '(12 14)
(info (triangle 3 4 5))        ; => '(6 12)
```

3-4-5 の直角三角形の面積が `6` になるのは、ヘロンの公式 `√(s(s-a)(s-b)(s-c))` が正しく動いた証拠。

**発展**: `match` の利点は、`info` が **完全な場合分け** になっていることを構造体名から検証できる点です。新しい図形(例: `polygon`)を追加したとき `match` に節を足し忘れると、評価時に「マッチ失敗」のエラーが出ます。

## 問題 3. `word-count`

### ヒント

`hash-update` の第 4 引数に初期値 `0` を渡すことで、「キーが未登場なら 0 から始めて `add1` する」を 1 行で書けます。

### 部分解答

```racket
(define (word-count words)
  (for/fold ([h (hash)]) ([w (in-list words)])
    (hash-update h w add1 0)))
```

### 完全解答

```racket
#lang racket

(define (word-count words)
  (for/fold ([h (hash)]) ([w (in-list words)])
    (hash-update h w add1 0)))

(word-count '("a" "b" "a" "c" "b" "a"))
;; => '#hash(("a" . 3) ("b" . 2) ("c" . 1))

;; 結果をソートした連想リストにしたい場合
(define (word-count/sorted words)
  (sort (hash->list (word-count words))
        > #:key cdr))

(word-count/sorted '("a" "b" "a" "c" "b" "a"))
;; => '(("a" . 3) ("b" . 2) ("c" . 1))
```

補足:

- 既定のハッシュ (`hash`) は **不変** で、更新は新しいハッシュを返す
- 書き換え版を使いたいときは `make-hash` + `hash-update!`
- 出現順を保ちたければハッシュではなく連想リスト + 既登場チェックで書く

```racket
;; mutable 版
(define (word-count! words)
  (define h (make-hash))
  (for ([w (in-list words)])
    (hash-update! h w add1 0))
  h)
```

`for/fold` の良さは「累算が 1 つ以上のとき」にも自然に広がるところ。たとえば `(累算=連想リスト, 累算=合計)` の 2 つを同時に回すケースでも同じ書き方でいけます。
