## 問題 1. 標準入力から読み取って素数判定

### ヒント

3 ステップに分解する。

1. `read-line` で 1 行読む
2. `string->number` で数値化、`#f` なら入力エラー
3. 2 から `√n` までで割り切れなければ素数

`string->number` は失敗すると `#f` を返すので、`with-handlers` は要りません。

### 部分解答

```racket
(define (prime? n)
  (cond
    [(< n 2) #f]
    [(= n 2) #t]
    [(even? n) #f]
    [else
     (for/and ([d (in-range 3 (add1 (integer-sqrt n)) 2)])
       (not (zero? (modulo n d))))]))
```

### 完全解答

```racket
#lang racket

(define (prime? n)
  (cond
    [(< n 2)   #f]
    [(= n 2)   #t]
    [(even? n) #f]
    [else
     (for/and ([d (in-range 3 (add1 (integer-sqrt n)) 2)])
       (not (zero? (modulo n d))))]))

(define (classify-line line)
  (define n (string->number (string-trim line)))
  (cond
    [(not (exact-integer? n))
     (displayln "error: not an integer")]
    [(prime? n) (displayln "prime")]
    [else       (displayln "composite")]))

(module+ main
  (let loop ()
    (define line (read-line))
    (unless (eof-object? line)
      (classify-line line)
      (loop))))
```

動作例:

```text
$ echo -e "7\n9\n2\nhello\n97" | racket prime.rkt
prime
composite
prime
error: not an integer
prime
```

`(string->number "hello")` が `#f` になるので、`exact-integer?` で弾けます。`with-handlers` の出番はありません。

## 問題 2. FizzBuzz を `/tmp/fizzbuzz.txt` に書き出す

### ヒント

`with-output-to-file` を使うと、ブロック内の `displayln` がそのままファイルに書き込まれます。

### 部分解答

```racket
(with-output-to-file "/tmp/fizzbuzz.txt" #:exists 'replace
  (lambda ()
    (for ([n (in-range 1 101)])
      (displayln
       (cond
         [(zero? (modulo n 15)) "FizzBuzz"]
         [(zero? (modulo n 3))  "Fizz"]
         [(zero? (modulo n 5))  "Buzz"]
         [else (number->string n)])))))
```

### 完全解答

```racket
#lang racket

(define (fizzbuzz n)
  (cond
    [(zero? (modulo n 15)) "FizzBuzz"]
    [(zero? (modulo n 3))  "Fizz"]
    [(zero? (modulo n 5))  "Buzz"]
    [else (number->string n)]))

(module+ main
  (with-output-to-file "/tmp/fizzbuzz.txt" #:exists 'replace
    (lambda ()
      (for ([n (in-range 1 101)])
        (displayln (fizzbuzz n)))))
  (displayln "written to /tmp/fizzbuzz.txt"))
```

確認:

```text
$ racket fizzbuzz.rkt
written to /tmp/fizzbuzz.txt
$ head -5 /tmp/fizzbuzz.txt
1
2
Fizz
4
Buzz
$ wc -l /tmp/fizzbuzz.txt
100 /tmp/fizzbuzz.txt
```

`#:exists 'replace` を指定しないと、ファイルが既にあるときにエラーになります。他の選択肢は `'append` / `'update` / `'truncate` など。

## 問題 3. `safe-number` で `or` を使う理由

### ヒント

`string->number` は成功時に数値、失敗時に `#f` を返します。Racket の `or` は **「偽でない最初の値」** を返す短絡評価。両者の「失敗値」が `#f` で一致するので、`or` で素直に繋げます。

### 部分解答

```racket
(define (safe-number s)
  (or (string->number s) #f))

(safe-number "42")   ; => 42
(safe-number "abc")  ; => #f
```

### 完全解答

**なぜ `or` が効くか** を言葉で:

- Racket では `#f` 以外はすべて **真** として扱われる(`0` も `""` も `'()` も真)。
- `(or a b)` は `a` が真ならそのまま `a`、`a` が偽(つまり `#f`)なら `b`。
- `string->number` が失敗すると `#f` を返すので、`(or (string->number s) #f)` は「成功なら数値、失敗なら `#f`」の意味に自然になる。

よく見ると、問題文のコード `(or (string->number s) #f)` は **`#f` 部分が冗長** です。`string->number` 自体が既に「成功 → 数値 / 失敗 → `#f`」を返しているので、

```racket
(define (safe-number s)
  (string->number s))
```

と書いても同じ挙動になります。問題文はあえて `or` の意味を確認させる練習問題なので、回答としては:

- `(or (string->number s) #f)` は **「失敗の時のデフォルト値を明示」** した形
- 別のデフォルトを使いたいときは `(or (string->number s) 0)` のように差し替えられる(これは `#f` と挙動が違う)
- `with-handlers` を使わない理由は「`string->number` が例外ではなく **値** で失敗を表現しているから」

とまとめられます。デフォルトを変えたい場面で `or` のパターンが効いてきます。

```racket
(define (safe-number/default s default)
  (or (string->number s) default))

(safe-number/default "hi" 0) ; => 0
(safe-number/default "7"  0) ; => 7
```

例外ではなく値で失敗を表すスタイルは、呼び出し側の分岐を `or` / `and` / `cond` に吸収できるので、Lisp 系では頻出です。
