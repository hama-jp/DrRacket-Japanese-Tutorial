## 問題 1. `double-all` を再帰で書く

### ヒント

再帰の型は「空リストはどうする / 非空リストは先頭と残りにどう分ける」の 2 択を書き切るだけです。

### 部分解答

```racket
(define (double-all xs)
  (if (null? xs)
      '()
      (cons (* 2 (car xs)) (double-all (cdr xs)))))
```

### 完全解答

```racket
(define (double-all xs)
  (cond
    [(null? xs) '()]
    [else (cons (* 2 (car xs)) (double-all (cdr xs)))]))

(double-all '(1 2 3 4))
;; => '(2 4 6 8)
(double-all '())
;; => '()
```

`if` 版と `cond` 版はどちらでも構いません。節が 3 つ以上になりそうなら最初から `cond` を書くほうが拡張に強いです。

## 問題 2. なぜ `double-all` は末尾再帰ではないか / 末尾再帰化する

### ヒント

「末尾再帰」= **再帰呼び出しの結果をそのまま返している** 形。`(cons ... (double-all ...))` のように再帰の外側に計算(`cons`)が残っているものは末尾再帰ではありません。

### 部分解答

`cons` のために「再帰が返ってきた後」も計算が残る → スタックが積まれる。末尾再帰化の定番は **累算引数(accumulator)に逆順で積み、最後に `reverse`** です。

```racket
(define (double-all-iter xs)
  (let loop ([xs xs] [acc '()])
    (if (null? xs)
        (reverse acc)
        (loop (cdr xs) (cons (* 2 (car xs)) acc)))))
```

### 完全解答

```racket
(define (double-all-iter xs)
  (let loop ([xs xs] [acc '()])
    (cond
      [(null? xs) (reverse acc)]
      [else (loop (cdr xs) (cons (* 2 (car xs)) acc))])))

(double-all-iter '(1 2 3 4))
;; => '(2 4 6 8)
```

補足:

- Racket は名前付き `let` や `letrec` に対して **末尾呼び出し最適化**(TCO)を保証しています。長いリストでもスタックを消費しません。
- `reverse` は `O(n)` ですが **1 回** しか走らないので、全体でも `O(n)`。
- より高階な書き方なら `for/list` もあります。こちらは内部的に累算引数方式で実装されています。

```racket
(define (double-all-for xs)
  (for/list ([x (in-list xs)]) (* 2 x)))
```

## 問題 3. `mystery` は何を計算しているか

### ヒント

`(car xs)` と `(cadr xs)` を足して 1 要素に畳み、`(cddr xs)` を後ろにつなぎ直しています。列の左 2 つを合計して 1 つにする操作を、長さが 1 になるまで続ける。

### 部分解答

```text
(mystery '(1 2 3 4 5))
= (mystery '(3 3 4 5))     ; 1+2
= (mystery '(6 4 5))       ; 3+3
= (mystery '(10 5))        ; 6+4
= (mystery '(15))          ; 10+5
= 15
```

実質は **リストの総和** です。`foldl + 0` や `(apply + xs)` と同じ結果になります。

### 完全解答

```text
> (mystery '(1 2 3 4 5))
15
> (mystery '(10 20))
30
> (mystery '())
0
> (mystery '(42))
42
```

比較のため他の書き方で同じ答えが出ることを確認:

```racket
(apply + '(1 2 3 4 5)) ; => 15
(foldl + 0 '(10 20))   ; => 30
```

「何をやっているか」は分かった上で、**この書き方がなぜ一般的でないか** も押さえておくと実用力が上がります。`cons` と `cddr` を組み合わせるのでリストの再割り当てが毎回発生し、`O(n^2)` になります。素直に 1 要素ずつ畳む `foldl` のほうが `O(n)` で速いです。再帰の型を学ぶ教材としては良いですが、本番コードでは `foldl` / `apply +` を選びましょう。
