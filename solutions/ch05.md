## 問題 1. `pow` の挙動を予想する

### ヒント

`pow` は「指数 `n` が偶数なら底を二乗して指数を半分にする」 を繰り返します。`b^n` の不変量を保ちながら `n` を指数的に減らす、繰り返し二乗法(exponentiation by squaring)の古典アルゴリズムです。

### 部分解答

たとえば `(pow 2 10)` は次のように展開されます。

```text
(pow 2 10)
= (pow 4 5)          ; even → (b*b, n/2)
= (* 4 (pow 4 4))    ; odd
= (* 4 (pow 16 2))
= (* 4 (pow 256 1))
= (* 4 (* 256 (pow 256 0)))
= (* 4 (* 256 1))
= 1024
```

乗算の回数は素朴な累乗の `O(n)` ではなく `O(log n)` になります。

### 完全解答

```text
> (pow 2 10)
1024
> (pow 3 5)
243
> (pow 5 0)
1
> (pow 2 16)
65536
```

不変量の確認として、評価の途中で「まだ計算すべき `b^n`」を手書きで追うと納得しやすいです。

| ステップ | 残り計算 | ここまで貯めた係数 |
| --- | --- | --- |
| `(pow 2 10)` | `2^10` | `1` |
| `(pow 4 5)` | `4^5` | `1` |
| `(pow 4 4)` | `4^4` | `4` |
| `(pow 16 2)` | `16^2` | `4` |
| `(pow 256 1)` | `256^1` | `4` |
| `(pow 256 0)` | `1` | `4 * 256` |
| 終了 | — | `1024` |

## 問題 2. `compose` で「3 乗して 1 加える関数」を 1 行で

### ヒント

`compose` は「右の関数を先に適用して、左の関数に渡す」 つまり `(compose f g)` = `λx. (f (g x))`。今回は **3 乗 → +1** の順なので、左に `add1`、右に `(λ(x) (* x x x))` を置きます。

### 部分解答

```racket
(define cube+1 (compose add1 (lambda (x) (* x x x))))
```

`add1` は `(lambda (x) (+ x 1))` の省略形で、組み込みで用意されています。

### 完全解答

```racket
(define cube+1 (compose add1 (lambda (x) (* x x x))))

(cube+1 2) ; => 9
(cube+1 0) ; => 1
(cube+1 -3) ; => -26
```

`compose` は可変長引数を取るので、もっと長い合成も同じ要領で書けます。

```racket
((compose symbol->string number->string add1) 1)
;; => "2"
```

## 問題 3. `make-counter-ex` を動かす / 拡張する

### ヒント

`lambda args` で書くと、呼び出し時のすべての引数が **リスト** として `args` に入ります。`(c)` なら `args = '()`、`(c 'reset)` なら `args = '(reset)`。

### 部分解答

```racket
(define (make-counter-ex)
  (define n 0)
  (lambda args
    (cond
      [(null? args)                (set! n (+ n 1)) n]
      [(equal? (car args) 'reset)  (set! n 0) 'reset-done]
      [(equal? (car args) 'get)    n])))
```

`'get` を足したバージョンで、「現在値を返すだけ」の読み取り API を追加できます。

### 完全解答

問題文のコードをそのまま動かすとこうなります。

```text
> (define c (make-counter-ex))
> (c)
1
> (c)
2
> (c 'reset)
'reset-done
> (c)
1
```

少し拡張して、`'get` を足す:

```racket
(define (make-counter-ex)
  (define n 0)
  (lambda args
    (cond
      [(null? args)              (set! n (+ n 1)) n]
      [(equal? (car args) 'reset) (set! n 0) 'reset-done]
      [(equal? (car args) 'get)   n]
      [else (error 'counter "unknown command: ~v" (car args))])))

(define c (make-counter-ex))
(c) (c) (c 'get) ; => 2
(c 'reset)
(c 'get)         ; => 0
(c 'foo)         ; => error: unknown command
```

`n` が `lambda args` の外側で `define` されているのがキモで、`c` に包まれた「私的な状態」として残り続けます。これが **クロージャ** です。
