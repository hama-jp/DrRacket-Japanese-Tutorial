## 問題 1. `foldr` で `my-map`

### ヒント

`foldr` の畳み込み関数は `(要素 累算) -> 新しい累算`。`map` なら要素に `f` を適用して、累算リストに `cons` するだけ。

### 部分解答

```racket
(define (my-map f xs)
  (foldr (lambda (x acc) (cons (f x) acc)) '() xs))
```

### 完全解答

```racket
(define (my-map f xs)
  (foldr (lambda (x acc) (cons (f x) acc)) '() xs))

(my-map add1 '(1 2 3))
;; => '(2 3 4)
(my-map (lambda (x) (* x x)) '(1 2 3 4))
;; => '(1 4 9 16)
```

`foldr` が「右から左に畳む」 ので、順序は標準の `map` と一致します。`foldl` で同じことをやると **逆順** になる点だけ注意(その場合は最後に `reverse` を付ける)。

## 問題 2. `foldr` で `my-filter`

### ヒント

畳み込み関数の中で、述語が真なら `cons`、偽なら累算を素通しするだけ。

### 部分解答

```racket
(define (my-filter pred xs)
  (foldr (lambda (x acc) (if (pred x) (cons x acc) acc)) '() xs))
```

### 完全解答

```racket
(define (my-filter pred xs)
  (foldr (lambda (x acc) (if (pred x) (cons x acc) acc)) '() xs))

(my-filter even? '(1 2 3 4 5 6))
;; => '(2 4 6)
(my-filter positive? '(-1 0 1 -2 3))
;; => '(1 3)
```

`my-map` と並べると、両方とも「要素を見て、累算リストに入れるか入れないか / 変形して入れるか」だけの違いです。`foldr` が **「リスト上の一般的な畳み込み」** の器であることが腹に落ちます。

発展として、両方を合成した `filter-map` も一行で書けます。

```racket
(define (my-filter-map f xs)
  (foldr (lambda (x acc)
           (let ([y (f x)])
             (if y (cons y acc) acc)))
         '() xs))

(my-filter-map (lambda (x) (and (odd? x) (* x x))) '(1 2 3 4 5))
;; => '(1 9 25)
```

標準の `filter-map` と同じ挙動です。

## 問題 3. `for/list` を `map` に書き直す

### ヒント

`for/list` のループ変数はそのまま `map` のラムダ引数に写せます。

### 部分解答

```racket
(map (lambda (x) (* 2 x)) '(1 2 3 4 5))
;; => '(2 4 6 8 10)
```

### 完全解答

```racket
(for/list ([x '(1 2 3 4 5)]) (* 2 x))     ; 元の書き方
;; => '(2 4 6 8 10)

(map (lambda (x) (* 2 x)) '(1 2 3 4 5))    ; map 版
;; => '(2 4 6 8 10)
```

使い分けの目安:

- 1 つのシーケンスを変換するだけなら `map` が簡潔
- 複数のシーケンスを同時に回す、添字を使う、条件でスキップする、といった要素が入ってくると `for/list` が読みやすい

```racket
;; 2 つのリストを同時に回す
(for/list ([a '(1 2 3)] [b '(10 20 30)]) (+ a b))
;; => '(11 22 33)

;; 添字付きで集める
(for/list ([x '(a b c)] [i (in-naturals)])
  (list i x))
;; => '((0 a) (1 b) (2 c))
```

## 問題 4. 学生の平均点

### ヒント

`map cadr` で点数だけを抜き出し、`apply +` で合計、`length` で件数。

### 部分解答

```racket
(define students '(("Reki" 80) ("Yui" 95) ("Ken" 70)))

(define (avg scores)
  (/ (apply + (map cadr scores)) (length scores)))

(avg students)
;; => 245/3
```

分数で返るのは Racket の整数除算が **厳密** だから。浮動小数がほしければ:

```racket
(exact->inexact (avg students))
;; => 81.66666666666667
```

### 完全解答

```racket
(define students
  '(("Reki" 80) ("Yui" 95) ("Ken" 70)))

(define (avg scores)
  (cond
    [(null? scores) (error 'avg "empty list")]
    [else (/ (apply + (map cadr scores)) (length scores))]))

(avg students)         ; => 245/3
(exact->inexact (avg students))
                       ; => 81.66666666666667

;; 空リスト対応と、平均を小数で欲しい両方を満たす版
(define (avg/inexact scores)
  (if (null? scores)
      0.0
      (exact->inexact (/ (apply + (map cadr scores)) (length scores)))))

(avg/inexact students) ; => 81.66666666666667
(avg/inexact '())       ; => 0.0
```

別解として `for/sum` で合計を取ると、`apply` を使わない書き方になります。

```racket
(define (avg2 scores)
  (exact->inexact
   (/ (for/sum ([s (in-list scores)]) (cadr s))
      (length scores))))
```

どちらも意味は同じ。好みで選んで構いません。
